# 示例简历片段

以下内容是匿名示例，用来演示 skill 的使用方式。

## Go 后端项目：在线题库系统

技术栈：Go、Gin、GORM、MySQL、Redis、Docker

项目描述：

实现一个面向学生的在线刷题系统，支持用户登录、题目列表、提交答案、判题结果记录和错题复习。

个人职责：

- 负责后端接口设计与实现
- 设计用户表、题目表、提交记录表
- 使用 Redis 缓存题目详情，减少重复查询
- 使用 Docker Compose 部署本地开发环境
- 编写基础接口文档，和前端同学联调

项目难点：

- 提交记录和错题本之间的数据一致性
- 热门题目详情接口的查询性能
- 前后端联调时错误码和返回结构不统一

## C++ 后端项目：简易 HTTP 服务器

技术栈：C++、Linux、Socket、epoll、线程池

项目描述：

实现一个 Linux 下的简易 HTTP 服务器，支持静态文件访问、连接管理和基础日志输出。

个人职责：

- 实现基于 epoll 的 IO 事件处理
- 实现线程池处理请求解析和响应生成
- 使用 RAII 管理连接和文件资源
- 处理客户端断开、请求格式错误等异常情况

项目难点：

- 连接生命周期管理
- 多线程下任务队列的安全访问
- 大量短连接下的资源释放和日志定位
